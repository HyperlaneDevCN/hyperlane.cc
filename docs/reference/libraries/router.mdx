# `Router` 库

希望编写一次合约并在多个链上部署的应用程序开发者应该考虑使用 [`Router`](https://github.com/hyperlane-xyz/hyperlane-monorepo/blob/v3/solidity/contracts/client/Router.sol) 模式。

在这种模式中,应用程序的合约实例部署在每个支持的链上。每个实例都知道其他链上实例的地址。这些实例使用 Hyperlane 与远程链上的实例进行信息和状态的通信。

<RouterDiagram />

开发者可以继承 `Router` 混入合约来使用这种模式。`Router` 是一个 [`MailboxClient`](./mailboxclient.mdx),它跟踪远程链上其他 `Router` 合约的地址。这使得 `Router` 可以直接向其他 `Router` 发送消息而无需指定地址。它还允许 `Router` 拒绝来自其他不受信任发送者的消息。
