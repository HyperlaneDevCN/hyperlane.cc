# 术语表

## Agent (代理)

Hyperlane [代理](../protocol/agents.mdx)是读取和写入Hyperlane智能合约状态的链下参与者。

代理的例子包括[验证者](../protocol/agents/validators.mdx)和[中继器](../protocol/agents/relayer.mdx)。

## Aggregation ISM (聚合式ISM)

[聚合式ISM](../protocol/ISM/aggregation-ISM.mdx)是一种跨链安全模块,它通过要求n个[ISM](glossary.mdx#interchain-security-module)中的m个验证特定的跨链消息来聚合多个ISM的安全性。

## Checkpoint (检查点)

检查点是一个(默克尔根,索引)元组,对应于特定时间点[邮箱](glossary.mdx#mailbox)增量默克尔树的状态。

[验证者](glossary.mdx#validator)的检查点签名用于[多重签名ISM](glossary.mdx#multisig-ism)。

## Collateral chain (抵押链)

对于[Warp路由](glossary.mdx#warp-route)而言,抵押链是[抵押代币](glossary.mdx#collateral-token)被存入Warp路由的链。

## Collateral token (抵押代币)

对于[Warp路由](glossary.mdx#warp-route)而言,抵押代币是在[抵押链](glossary.mdx#collateral-chain)上存入以在远程[合成链](glossary.mdx#synthetic-chain)上创建包装代币的代币。

## Default ISM (默认ISM)

当消息接收方没有指定自己的ISM时,用于验证入站消息的[跨链安全模块](glossary.mdx#interchain-security-module)。

## Domain (域)

Hyperlane协议用来确定消息来源和目的地的特定链的唯一标识符。

可能与EVM链ID相同,但并非总是如此。查看[domains](domains/)获取已知的Hyperlane域ID列表。

## Interchain accounts (ICA) (跨链账户)

一个允许用户进行跨链智能合约调用的[中间件](glossary.mdx#middleware)智能合约。

例如,DAO可以使用跨链账户在远程链上拥有合约。

查看[跨链账户](../reference/applications/interchain-account.mdx)获取更多信息。

## Interchain gas paymaster (跨链gas支付)

由[中继器](glossary.mdx#relayer)部署的智能合约,用于在源链上接受目标链消息传递的支付。

查看[跨链gas支付](../protocol/interchain-gas-payment.mdx)获取更多信息。

## Interchain queries (IQS) (跨链查询)

一个允许用户进行跨链视图调用的[中间件](glossary.mdx#middleware)智能合约。

例如,智能合约可以使用跨链查询来查看远程链上的预言机汇率或代币余额。

## Interchain Security Module (ISM) (跨链安全模块)

[跨链安全模块](../protocol/ISM/modular-security.mdx)(ISM)是为Hyperlane的跨链[邮箱](../protocol/mailbox.mdx)提供安全保障的智能合约。

ISM负责验证在目标链上传递的跨链消息确实是在源链上发送的。

## Mailbox (邮箱)

作为最重要的Hyperlane智能合约,[邮箱](../protocol/interchain-gas-payment.mdx)提供了一个开发者可以用来[发送](./messaging/send.mdx)和[接收](./messaging/receive.mdx)跨链消息的接口。

## Middleware (中间件)

一个发送和接收消息并提供面向开发者API的智能合约。开发者应该使用这个API而不是直接与[邮箱](glossary.mdx#mailbox)交互。

中间件的例子包括[跨链账户-ica](glossary.mdx#interchain-accounts-ica)和[跨链查询-iqs](glossary.mdx#interchain-queries-iqs)。

## Multisig ISM (多重签名ISM)

[多重签名ISM](../protocol/ISM/multisig-ISM.mdx)是一种[跨链安全模块](glossary.mdx#interchain-security-module),它使用n个[验证者](glossary.mdx#validator)中的m个签名来验证特定的跨链消息。

## Permissionless Interoperability (无许可互操作性)

任何人都可以将Hyperlane部署到任何链上,无论是第一层、rollup还是应用链,使该链能够无缝通信。Hyperlane协议的开放设计被称为无许可互操作性(PI)。由Hyperlane核心团队以外的机构部署的链被称为"PI链"。

## Relayer (中继器)

[中继器](../protocol/agents/relayer.mdx)是负责将消息从源链传递到目标链的Hyperlane[代理](glossary.mdx#agent)。

中继器是不受信任的,任何人都可以运营中继器。

## Routing ISM (路由ISM)

[路由ISM](../protocol/ISM/routing-ISM.mdx)是一种[跨链安全模块](glossary.mdx#interchain-security-module),它根据正在传递的消息委托给不同的ISM。

例如,路由ISM可以根据发送消息的源链使用不同的ISM。

## Synthetic chain (合成链)

对于[Warp路由](glossary.mdx#warp-route)而言,合成链是铸造包装代币的链。这些包装代币由锁定在[抵押链](glossary.mdx#collateral-chain)上的[抵押代币](glossary.mdx#collateral-token)支持。

## Validator (验证者)

[验证者](../protocol/agents/validators.mdx)是负责验证源链发送消息的Hyperlane[代理](glossary.mdx#agent)。

[多重签名ISM](glossary.mdx#multisig-ism)使用验证者签名为入站消息提供安全保障。

## Warp Route (Warp路由)

[Warp路由](applications/warp-routes.mdx#overview)是Hyperlane对代币桥接概念的实现,允许你通过Hyperlane将任何类ERC20资产无许可地转移到任何链。

了解更多关于如何[部署Warp路由](../guides/deploy-warp-route.mdx)的信息。
