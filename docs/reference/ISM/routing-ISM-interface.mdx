# 路由模块

开发者可以使用 `RoutingISM` 将消息验证委托给不同的 ISM。这使得开发者可以根据消息内容或应用上下文更改安全模型。

## 接口

`RoutingISM` 必须实现 `IRoutingIsm` 接口。

```solidity file=<rootDir>/node_modules/@hyperlane-xyz/core/contracts/interfaces/isms/IRoutingIsm.sol
```

## 配置

hyperlane-monorepo 包含了一个可以直接使用的 `RoutingISM` 实现 `DomainRoutingIsm`，应用开发者可以部署它并指定所需的配置。

这个 ISM 只是根据消息的源链切换安全模型。一个简单的用例是为每个链使用不同的 [Multisig ISM](multisig-ISM-interface.mdx) 验证者集合。

未来，你可以设想 `DomainRoutingIsm` 根据源链使用的共识协议类型路由到不同的基于轻客户端的 ISM。

## 自定义

hyperlane-monorepo 包含了一个抽象的 `RoutingISM` 实现，应用开发者可以基于它进行扩展。

开发者只需要实现 `route()` 函数即可。

通过创建自定义实现，应用开发者可以根据应用需求定制 `RoutingISM` 提供的安全性。

例如，自定义实现可以根据消息内容或接收消息的应用状态来更改安全模型。
