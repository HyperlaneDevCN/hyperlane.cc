# 乐观ISM

优先考虑安全性而非活跃性

:::warning
`OptimisticISM` 即将推出，目前尚未实现。此页面仅供参考，具体细节可能会随着设计的成熟而变化。
:::

`OptimisticISM` 编码了由 [Optics](https://docs.celo.org/protocol/bridge/optics) 首创并被 [Synapse](https://docs.synapseprotocol.com/docs/RFQ/Security/#proofs)、[Nomad](https://docs.nomad.xyz/the-nomad-protocol/verification-mechanisms/optimistic-verification) 和 [Connext](https://blog.connext.network/optimistic-bridges-fb800dc7b0e0) 采用的乐观验证安全模型。

`OptimisticISMs` 在消息验证后依赖于一个欺诈窗口，在此期间，配置在 `OptimisticISM` 上的 `m` 个中的 `n` 个 **观察者** 有机会将消息标记为欺诈，从而防止其被发送给接收者。

**这种安全模型优先考虑安全性而非活跃性；** 增加的消息延迟允许在不显著增加 Gas 成本的情况下，增加第二层安全性，即观察者。
