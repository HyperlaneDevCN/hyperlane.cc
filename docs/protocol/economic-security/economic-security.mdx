# 经济安全

:::info

Hyperlane 致力于使欺诈成本高昂。质押和惩罚即将到来，但尚未实施。本页面仅供参考，具体细节可能会随着设计的成熟而变化。

:::

质押和惩罚允许将经济安全添加到依赖验证者签名的 ISM 中，包括 [多重签名 ISM](/docs/protocol/ISM/multisig-ISM.mdx) 和乐观 ISM。

Hyperlane 验证者可以选择参与 Hyperlane 的质押协议。稍后，如果这些验证者试图伪造或审查跨链消息，他们的质押以及任何委托给他们的质押都可能会被惩罚。

依赖于这些验证者签名的 [跨链安全模块](/docs/protocol/ISM/modular-security.mdx) 将从质押和惩罚提供的额外经济安全中受益。

### 可验证的欺诈证明

与许多其他跨链通信协议不同，Hyperlane 的惩罚协议使用 **可验证的欺诈证明**。

这意味着 Hyperlane 协议能够在没有可信方参与的情况下验证验证者是否签署了欺诈性检查点。

这是可能的，因为验证者所提供的质押与他们所证明的状态（即邮箱默克尔根）在 **同一链** 上。惩罚智能合约可以将验证者的签名与最新的根进行比较，并进行一些复杂的默克尔树操作，以确认验证者签署的检查点是否存在欺诈行为。

在许多其他跨链通信协议中，验证者的质押通常位于与跨链消息来源不同的 **链** 上。

这意味着，为了惩罚欺诈的验证者，他们的质押必须通过相同的消息传递协议传递到质押所在的链。你能看到这个问题吗？发生欺诈的同一验证者集是传递欺诈证据的机制，这样会有什么问题呢？

Hyperlane 不希望允许这种可能性，因此在 Hyperlane 中，验证者必须将其绑定的质押保留在他们所验证的原始链上。这意味着欺诈证明是可以无信任验证的。用于惩罚的欺诈记录存在于与要惩罚的质押相同的环境中，消除了欺诈证明过程中的错误空间。
