# 如何将您的Hyperlane部署推向生产环境

要将您的Hyperlane部署推向生产环境，请按照以下步骤设置您链的核心部署并配置warp路由部署。

### 1. 核心部署

首先，您需要设置并使您的Hyperlane部署的核心组件适合生产环境。

- [更新Mailbox默认ISM](/guides/update-mailbox-default-ism.mdx)：更新与Mailbox合约关联的链间安全模块(ISM)。ISM作为关键的安全层，负责验证链间的消息。确保默认ISM设置符合您的安全要求。
- [转移Mailbox所有权](/guides/transfer-mailbox-ownership.mdx)：将Mailbox合约的所有权转移给生产就绪的所有者账户。这一步通过确保只有受信任的方才能控制其配置来保障Mailbox的安全。

### 2. Warp路由部署

- [移除受信任中继器](/guides/remove-trusted-relayer.mdx)：移除任何可能最初配置的测试或开发中继器。在生产环境中，确保只使用受信任、安全的中继器，以防止未授权的消息中继。
- [转移Warp路由所有权](/guides/transfer-warp-route-ownership.mdx)：将Warp路由的所有权转移给指定的生产环境所有者。这一步确保只有受信任的方才能控制关键的Warp路由设置，如ISM配置、验证器选项和中继器设置。
